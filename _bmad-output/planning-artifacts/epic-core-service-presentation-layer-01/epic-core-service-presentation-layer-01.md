# Epic: Core Service Presentation Layer

**Epic Owner:** Product Management
**Created:** 2026-01-22

## Overview

This epic introduces a presentation (API/controller) layer for the core service, exposing RESTful endpoints for tenant and building management, and handling promotion submissions. It also covers missing infrastructure for the building information module and global exception handling improvements.

## Goals

- Expose RESTful endpoints for tenant and building operations
- Ensure all endpoints are protected, validated, and documented
- Complete infrastructure for all core modules
- Standardize error handling across the service

## Stories

- S-core-service-01-presentation-endpoints.md
- S-core-service-02-building-info-infra.md
- S-core-service-03-global-exception-filter.md

## Acceptance Criteria

- All endpoints are implemented, validated, and documented
- Building information module has full infrastructure
- Global exception filter is in place and domain exceptions are standardized
